The Q for Eclipse is a plugin that better integrates the development and building processes by allowing users to run Maven commands inside of the Eclipse IDE.

Installing and Uninstalling the Eclipse Integration Plugin

Pre-requisites

  • Eclipse 3.3 or greater, which you can find at http://download.eclipse.org/eclipse/downloads
  • Java 5, which you can find at http://java.sun.com
    Java 5 must be the JDK version, not JRE. If you experience build errors, even if the JDK is first in your path, start Eclipse with "-vm c:\%JAVA_HOME%\bin\javaw.exe" , where JAVA_HOME is set to your JDK installation directory. This option can be put in the eclipse.ini file and then you can start Eclipse from your menu or shortcut.

Installing Q for Eclipse

  1. On the menu bar, click Help > Software Updates > Find and Install. An installation wizard opens.
  2. In the Feature Updates window:
    • Select Search for new features to install if this is the first time you have installed the plugin.
    • Select Search for updates if you already have a version of the plugin installed and are just performing an upgrade.
  3. On the Update sites to visit page, click the New Remote Site button (located on the right side). A small pop-up appears, allowing you to set up a bookmark for the site from which you will download the Q for Eclipse Integration Plugin.
  4. Enter a name or alias for the site from which you want to download the plugin (for example, “Q for Eclipse”). Then, enter the URL, http://q4e.googlecode.com/svn/trunk/updatesite/, for the download site. If you are behind a firewall that requires a proxy you must setup a proxy in Eclipse to reach the remote site. To setup a proxy go to Window > Preferences and in the new window select Network Connections located under General in the left hand expandable menu to establish a Manual Proxy Configuration.
  5. Click OK to add the new site to the list of sites in the window of the wizard.
  6. Click the box to select the site you just added, then click Finish. Eclipse then searches the site for available features you can install and and displays them in hierarchical tree structure on a Search Results page.
  7. On the Search Results page, click to select the box to the left of the feature(s) you want to install (including the Q for Eclipse plugin). Then, below the list of features, click to select Show the latest version of a feature only.
  8. Click Next to display the Feature License screen of the wizard. The Feature License screen shows a list of the features you selected that have license agreements you must accept before installing them. Below the list window, click the first option to indicate you accept the license agreement(s), then click Next to move to the Installation screen of the wizard.
  9. The Installation screen lists all of the features/plugins you have chosen to install, along with the corresponding directories into which they will install:
    • If you want to accept the default destination directories shown for the features you selected,
      1. Click Finish to download the features/plugins.
      2. After downloading, a Feature Verification screen will appear.
    • If you want to change the default destination directories shown for the features you selected,
      1. Click to highlight the feature whose destination you want to change.
      2. Click the Change Location button, specify the new destination, then click OK.
      3. After downloading, a Feature Verification screen will appear.
  10. On the Feature Verification screen, verify that you want to install the features you selected. To install all of the features, click Install All.
  11. When installation is complete, a window appears asking you whether you would like to restart Eclipse. Click Yes to restart so the new features will take effect.

Verifying Installation

  1. After Eclipse has been restarted, click on Help > About Eclipse SDK.
  2. On the pop-up, click Plug-in Details. A list of all plugins installed on Eclipse will appear.
  3. Scroll down and look for the Maestro Eclipse Integration Plugin you just installed by looking for the Provider name of DevZuz.

Uninstalling the Eclipse Integration Plugin

Uninstalling the Plugin

The uninstall process completely removes the Eclipse Integration Plugin from your computer.

  1. From inside Eclipse, select Help > Software Updates > Manage Configuration.
  2. A list of plugins is listed in the left pane of the Product Configuration window. Select Q for Eclipse and right-click.
  3. Click on Uninstall, and the system automatically uninstalls the version you selected.
  4. Repeat this operation for each plugin you want to remove. Most likely you will want to remove the enabling features that were installed as part of the Q for Eclipse Integration Plugin as well: Apache Commons Collections, Prefuse, and Apache Maven.
    Uninstalling the plugin will require you to restart Eclipse. Restart Eclipse for the changes to take effect.

Returning to a Previous Installation

Under Eclipse's Manage Configuration page, there are several very useful options, such as View Installation History and Revert to Previous. View Installation History shows a log of all previous installations, and Revert to Previous lets you choose to return to a specific, previous installation (for example, the installation just prior to an upgrade). To return to a previous installation:

  1. From inside Eclipse, select Help > Software Updates > Manage Configuration. The Product Configuration screen appears, showing a list of components on the left side and a list of links on the right side.
  2. On the right side of the Product Configuration screen, click on the link for Revert to Previous to see a list of the previous installations organized by date and time.
  3. Click to select the installation to which you want to return.
  4. Select Revert to Previous. The system automatically returns to the installation and configuration that corresponds to your selection.
    This option is very useful if you have a problem with the upgrade procedure and want to return to the version you had before.
