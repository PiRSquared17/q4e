Licenses/SCM/Organization/Issue Management

This section provides description and instructions on modifying the License, SCM, Organization, and Issue Management sections.

Modifying the values in the pom.xml does not automatically save the changes. You need to save the POM file to make the changes take effect.

Modifying Licenses

To add a license:

  1. Click New.
  2. Fill in the values for the fields in the pop-up window.
  3. Click OK.

To edit a license

  1. Click Edit.
  2. Modify the values for the fields in the pop-up window.
  3. Click OK.

To remove a license, select the license from the list and click Remove.

Source Code Management

The Source Code Management (SCM) contains configurations for the address of the Subversion repository of the project.

To modify the configuration, supply the values for the fields to be changed and save the file for the changes to take effect.

Organization Information

The Organization section of the POM contains the basic information of the organization that produced the project.

To modify the name and URL of the organization, supply the values for the fields and save the file for the changes to take effect.

Issue Management

The Issue Management section contains the information for the issue or bug tracking system of the project.

To modify the system and URL of the issue tracking system, supply the values for the fields to be changed and save the file for the changes to take effect.
